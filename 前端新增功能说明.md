# Dashboard 数据概览 + 知识库管理页 - 功能说明

## 概述

「数据概览」页面作为系统首页，用户可以一目了然地看到 RAG 流水线各阶段的数据状态，切换查看各阶段的完整文档列表，并对文档进行搜索、删除和批量管理操作。

## 页面结构

页面分为两个区域：

### 1. 统计卡片区域（可切换）

顶部展示 5 张统计卡片，对应 RAG 流水线的 5 个阶段：

| 卡片 | 数据来源 API | 选中时显示的列表内容 |
|------|-------------|--------------------|
| 已导入文档 | `GET /documents?type=loaded` | 文件名、加载方式、页数、分块数、时间 |
| 已分块文档 | `GET /documents?type=chunked` | 文件名 |
| 已嵌入文档 | `GET /list-embedded` | 文件名、嵌入模型、提供者、维度、时间 |
| 向量库集合 | `GET /collections` | 集合名称 |
| 检索记录 | `GET /search-results` | 检索记录名、时间 |

点击卡片切换下方文档列表视图，选中卡片带有对应颜色的边框高亮。切换时自动重置搜索和选择状态。

### 2. 文档列表区域（含管理功能）

根据当前选中的卡片，展示该阶段的完整文档列表，包含序号和对应字段。
默认显示「已导入文档」列表。

#### 搜索筛选

列表标题栏右侧提供搜索输入框，输入关键词即可按文件名实时过滤当前列表。

#### 单条删除

每行末尾有「删除」按钮，点击后弹出确认对话框，确认后调用后端删除 API。

#### 批量删除

- 表头第一列为全选 checkbox
- 每行第一列为选择 checkbox
- 选中项后，列表顶部出现红色「批量删除 (N)」按钮
- 点击后弹出确认对话框，确认后逐条删除并刷新数据

#### 删除 API 对应关系

| Tab | 删除 API |
|-----|----------|
| 已导入文档 | `DELETE /documents/{doc_name}?type=loaded` |
| 已分块文档 | `DELETE /documents/{doc_name}?type=chunked` |
| 已嵌入文档 | `DELETE /embedded-docs/{doc_name}` |
| 向量库集合 | `DELETE /collections/chroma/{collection_name}` |
| 检索记录 | 无删除 API，不显示删除功能 |

## 技术实现要点

1. **并发请求**：使用 `Promise.all` 同时发起 5 个 API 请求，减少页面加载时间
2. **加载状态**：请求期间显示骨架屏动画（`animate-pulse`），提升用户体验
3. **容错处理**：每个 API 请求单独 `.catch()`，某个接口失败不影响其他数据展示
4. **视图切换**：点击卡片通过 `activeTab` 状态切换下方文档列表，无需路由跳转
5. **选中高亮**：使用固定 `border-2` 实现选中状态，未选中时 `border-transparent` 占位，避免布局跳动
6. **搜索筛选**：前端关键词过滤，基于 `searchKeyword` 状态对列表做 `.filter()`
7. **批量选择**：`selectedItems` 数组管理选中项，全选/取消全选联动
8. **删除确认**：模态遮罩 + 居中卡片，区分单条/批量删除提示，红色警示按钮
9. **URL 编码**：删除请求中对文件名使用 `encodeURIComponent` 防止特殊字符问题
10. **无新增依赖**：纯 Tailwind CSS 实现所有样式，不引入任何额外库

## 如何运行

确保后端服务已启动（默认 `http://localhost:8001`），然后：

```bash
cd frontend
npm run dev
```

浏览器访问 `http://localhost:5174` 即可看到 Dashboard 页面。
